Selección de Datos

Identificar fuentes de datos relevantes para el modelo de priorización

Fuentes de Datos:
  • Registros históricos de usuarios atendidos en la Casa de Justicia
  • Bases de datos de entidades remitentes
  • Información demográfica y socioeconómica
  • Registros de víctimas y poblaciones vulnerables
Variables Clave:
  • Edad, género, nacionalidad
  • Estrato socioeconómico
  • Entidad remitente
  • Condición de víctima
Limpieza de Datos

Corregir errores y gestionar valores faltantes para garantizar calidad de datos

Procesos de Limpieza:
  • Identificación y corrección de valores atípicos
  • Manejo de datos faltantes mediante imputación
  • Eliminación de registros duplicados
  • Normalización de formatos de fecha y texto
Validación de Calidad:
  • Verificación de integridad referencial
  • Comprobación de rangos válidos para variables numéricas
  • Validación de categorías para variables categóricas
Transformación

Crear nuevas características y normalizar datos para optimizar el rendimiento del modelo

Ingeniería de Características:
  • Codificación de variables categóricas (One-Hot Encoding)
  • Normalización y estandarización de variables numéricas
  • Creación de variables derivadas (edad en rangos, índices compuestos)
  • Binning de variables continuas cuando sea apropiado
Preparación para ML:
  • División de conjuntos de entrenamiento y prueba
  • Balanceado de clases si es necesario
  • Selección de características más relevantes
Documentación

Documentación del Proceso:

  • Justificación de decisiones de limpieza y transformación
  • Métricas de calidad antes y después del procesamiento
  • Scripts y código utilizado en cada etapa
Metadatos y Linaje:

  • Diccionario de datos actualizado
